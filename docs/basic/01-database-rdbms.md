# 01. Database와 RDBMS

## Database란 무엇인가

Database는 여러 데이터를 일정한 규칙으로 저장하고 다시 찾기 위한 공간입니다. 파일 하나에 JSON을 저장할 수도 있지만, 데이터가 많아지고 여러 기능이 동시에 접근하면 검색, 관계, 중복, 안전한 변경을 관리할 도구가 필요합니다.

Database Management System(DBMS)은 이 저장과 조회를 담당하는 소프트웨어입니다.

## 관계형 데이터베이스

RDBMS는 데이터를 테이블로 나눕니다.

```text
members
+----+--------+-------------------+
| id | name   | email             |
+----+--------+-------------------+
|  1 | 민지   | minji@example.com |
+----+--------+-------------------+
```

- table: 같은 종류의 데이터를 모은 구조
- row: 한 개의 데이터
- column: 모든 행이 공유하는 속성
- primary key: 각 행을 구분하는 고유한 값

## SQLite를 사용하는 이유

SQLite는 별도 서버 없이 하나의 파일에 관계형 데이터를 저장합니다. SQL과 테이블 구조를 익히는 데 집중할 수 있어 첫 RDBMS 실습에 적합합니다.

이 과정에서는 Node.js 기본 `node:sqlite` 모듈로 SQLite 파일을 만들고 SQL을 실행합니다.
