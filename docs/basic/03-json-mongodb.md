# 03. JSON과 MongoDB

## 문서와 컬렉션

MongoDB는 JSON과 비슷한 BSON 문서를 저장합니다. JavaScript 객체의 중첩 객체와 배열 모양을 비교적 자연스럽게 유지할 수 있습니다.

```json
{
  "isbn": "978-00-0001",
  "title": "데이터를 배우는 시간",
  "author": {
    "name": "김데이터"
  },
  "categories": ["database", "beginner"],
  "inventory": {
    "stock": 3,
    "location": "A-01"
  }
}
```

- document: 객체 한 개에 해당하는 저장 단위
- collection: 같은 용도의 문서 모음
- `_id`: MongoDB가 각 문서를 구분하는 고유 값

## 포함과 참조

항상 책과 함께 읽는 작은 저자 정보는 책 문서 안에 포함할 수 있습니다. 여러 문서가 공유하고 독립적으로 자주 바뀌는 정보는 별도 컬렉션에 두고 ID로 참조할 수 있습니다.

정답은 하나가 아닙니다. 어떤 화면에서 함께 읽는지, 데이터가 얼마나 자주 바뀌는지, 배열이 계속 커지는지를 함께 판단합니다.

## 인덱스

인덱스는 특정 필드로 문서를 빠르게 찾도록 돕습니다. 대신 저장 공간을 사용하고 쓰기 작업 비용이 늘어납니다. 자주 검색하거나 고유해야 하는 필드부터 선택합니다.

## 확인하기

1. 중첩 객체와 배열이 많은 데이터에 문서 구조가 편리한 이유는 무엇인가요?
2. 저자 정보를 포함할지 참조할지 결정할 때 무엇을 확인해야 하나요?
3. 모든 필드에 인덱스를 만들지 않는 이유는 무엇인가요?
